# Bao la Kiswahili 先攻・後攻差研究

Version: 0.3.0
更新日: 2026-07-15

## 1. 研究目的

Bao la Kiswahiliというゲームに、ゲーム開始時の先攻・後攻による有利不利が存在するかを、コンピュータ自己対局によって段階的に検証する。

本研究は、このリポジトリのルール実装とAIを用いた実験研究であり、Baoの理論的な先手必勝・後手必勝を証明するものではない。AI、探索深度、評価関数、序盤生成法への依存を検証しながら、観測された差の一般性を高める。

この実装では標準初期状態の`player: 0`がSouthであるため、Southをゲーム開始時の先攻、Northを後攻として扱う。

## 2. これまでの結果

### 2.1 予備調査

hard対normal、legacy評価、深度2の100局ではSouth 48勝、North 52勝だった。South勝率の95% Wilson信頼区間は38.5%〜57.7%であり、明確な先後差は確認できなかった。

### 2.2 標準初期局面の決定的自己対局

同一hard AI、bao評価、phase2探索、深度2で標準初期局面から1,000局を実行した結果、South 0勝、North 1,000勝、平均48手だった。

ただし、異なるseedでも着手系列ハッシュが一致したため、これは1,000個の独立局面ではなく、同一の決定的対局系列を反復した結果である。したがって、Bao一般の後攻必勝を意味しない。

### 2.3 深度別の標準初期局面診断

| 深度 | 勝者 |
| ---: | --- |
| 1 | 先攻South |
| 2 | 後攻North |
| 3 | 後攻North |
| 4 | 後攻North |

勝者が探索深度に依存しており、AIの読みの深さが観測結果へ影響することが確認された。

### 2.4 ランダム局面からの手番価値

namuaとmtajiのランダム局面を用いた1,200局では、namua全体の手番側勝率は48.7%でほぼ均衡、mtaji全体では手番側が57.3%勝利した。

この結果はゲーム開始時の先攻差ではなく、生成局面で現在手番を持つ側の価値を示す。Baoではphaseによって手番価値が変わる可能性がある。

### 2.5 ゲーム開始時の先攻・後攻差

標準初期局面から2、4、6、8、12手を一様ランダムに進め、その後を同一hard AI、bao評価、phase2探索、深度2で継続した1,000局の結果は次のとおりだった。

| ランダム序盤 | 先攻South | 後攻North | 先攻勝率 |
| ---: | ---: | ---: | ---: |
| 2手 | 62 | 138 | 31.0% |
| 4手 | 87 | 113 | 43.5% |
| 6手 | 103 | 97 | 51.5% |
| 8手 | 118 | 82 | 59.0% |
| 12手 | 91 | 109 | 45.5% |
| 合計 | 461 | 539 | 46.1% |

総合では後攻Northが53.9%勝利した。先攻South勝率の95%信頼区間は43.0%〜49.2%で、今回の条件では弱いが統計的に明確な後攻優位が観測された。

一方、序盤手数によって優位方向が反転している。したがって、固定的な後攻ハンディというより、序盤の分岐と局面構造が勝率へ強く影響している可能性が高い。

## 3. 現時点の結論

> 一様ランダムな序盤分岐と深度2 AIを用いた1,000局では、ゲーム開始時の後攻が53.9%勝利した。ただし、結果は探索深度と序盤生成条件に敏感であり、Bao一般に固定的な後攻優位があると断定する段階ではない。

現時点では、次の仮説を比較する必要がある。

1. Baoには小さな後攻優位がある。
2. 深度2の評価・探索が後攻に有利な戦略を生む。
3. 一様ランダム序盤が現実的でない局面を多く生成している。
4. 特定の初手または応手が総合勝率を左右している。
5. South/Northの座標変換や評価処理に非対称性がある。

## 4. 実験1〜5の短時間スクリーニング（未完了）

各条件はまず40局、10局×4バッチで異なるseedを使用する。11条件の合計は440局である。安定した結果や重要な差が見られた条件だけを後続で200局、必要に応じて1,000局へ拡張する。

2026-07-15現在、`artifacts/first-player-suite/screening-2026-07/summary.json`は作成されておらず、この節は実験計画である。従来計画の200局×11条件とMCTS 400 iterations／80手プレイアウトは約50〜120時間規模になったため保留し、通常実行しない`full-2026-07`プロファイルとして分離した。

短時間版は最大対局長を120手、MCTSを12 iterations／16手プレイアウトとする。代表局の実測ではphase2深度1が約0.7秒、深度2が約6.5秒、深度3が約2.8秒、深度4が約47秒、MCTS 12 iterationsの3局が約60秒だった。局面による変動を考慮した全体見積もりは約80〜150分である。

短時間MCTSはアルゴリズム差の初期兆候を確認するための低強度条件であり、元の400 iterations条件の代替結果とは扱わない。

### 実験1: 探索深度別の再現性

深度1、2、3、4を比較する。序盤は8手の一様ランダム、評価はbao、探索はphase2とする。

目的は、観測された後攻優位が浅い探索に固有かを確認することである。

### 実験2: 序盤生成方式の比較

次の3方式を比較する。

- `uniform`: 全合法手から一様ランダム
- `top3`: AI評価上位3手から一様ランダム
- `softmax`: AI評価に応じた確率選択

目的は、非現実的な悪手を含む一様ランダム化が結果を歪めていないかを確認することである。

### 実験3: 初手別勝率

実験1、2、5の全対局で、Southの初手を記録し、初手ごとの対局数、先攻勝率、後攻勝率を集計する。

目的は「先攻そのものが不利」なのか、「一部の初手が先攻勝率を押し下げている」のかを区別することである。

### 実験4: South/North鏡像対称性監査

ランダムに生成した到達可能局面200件について、South/North交換と左右反転を施した鏡像局面を作り、次を比較する。

- 合法手集合
- legacy評価
- bao評価

目的は、ルールエンジンや評価関数の座席依存を検出することである。監査結果は合否を断定するだけでなく、不一致局面の詳細を保存する。

### 実験5: 評価方式・探索方式の比較

次を比較する。

- legacy評価 + phase2探索
- bao評価 + phase2探索
- bao-v2評価 + phase2探索
- bao評価 + MCTS

目的は、後攻優位が特定の評価関数または探索アルゴリズムに依存するかを確認することである。

## 5. 実行構成

ローカル実験の運用手順:

```text
tools/experiments/README.md
```

再開可能なローカルランナー:

```text
tools/experiments/run-first-player-research.js
```

条件別の対局ランナー:

```text
tools/first-player-experiment-suite.js
```

鏡像対称性監査:

```text
tools/first-player-symmetry-audit.js
```

重い自己対局はGitHub Actionsでは実行しない。各局はバッチseedから導出した独立seedを持ち、1局終了ごとにpartial JSONへ原子的に保存する。中断後は同じコマンドで次の未完了局から再開する。10局ごとの正式なバッチJSONを区切りの記録とし、`progress.json`には現在の条件、完了局数、完了バッチの集計を保存する。

予定成果物は`artifacts/first-player-suite/screening-2026-07/`に条件別バッチ、`progress.json`、`symmetry.json`、`summary.json`として保存する。全条件完了時は集計を自動実行し、必要な44バッチと440局が揃わない場合は失敗させる。

予定対局数は11条件×40局＝440局である。実験3の初手別集計はこれらの対局から同時に取得する。

## 6. 統計上の扱い

- 主指標はゲーム開始時の先攻South勝率とする。
- 条件ごとに95% Wilson信頼区間を計算する。
- 40局の結果は粗いスクリーニングとして扱い、最終結論には200局、必要に応じて1,000局以上へ拡張する。
- 同一条件内でもseedまたはバッチ間のばらつきを確認する。
- 多数の条件を比較するため、探索的結果と事前に定めた主仮説を区別する。
- 統計的有意差だけでなく、50%からの差と勝率差を効果量として報告する。

## 7. 保留事項

人間の熟練者対局データとの比較は重要だが、現時点では利用可能な対局データを確保できていないため保留する。

将来データを入手できた場合は、プレイヤー実力差、大会形式、投了、時間切れ、先後決定方法を考慮してAI結果と比較する。

## 8. 研究上の限界

- 使用するルールはこのリポジトリの実装に限定される。
- AIは人間の熟練者を完全に再現しない。
- ランダム序盤は実戦の開局分布と一致するとは限らない。
- 深度や評価関数が変わると勝率が変化する可能性がある。
- 鏡像変換の定義自体もBaoの盤面座標規約に照らして検証する必要がある。

最終的な主張は、複数探索方式、複数評価方式、対称性監査、局面別解析を組み合わせて更新する。
